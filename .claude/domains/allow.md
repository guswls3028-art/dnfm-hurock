# Domain: allow (허락 페이지)

`allow.dnfm.kr` — 인터넷 방송인 "허락" 님의 방송/이벤트 페이지. 던파 모바일 스트리머. **이 repo (`dnfm-allow`) 의 단일 사이트.**

## 1. 본업

**허락님의 방송 + 시청자 참여 이벤트 운영 공간.** 사용자(방장) 가 친분 서비스로 제공.

핵심 기능군:
1. **방송 페이지** — 라이브 채널, 다시보기, 공지, 프로필.
2. **참여형 이벤트** — 시청자가 직접 글 쓰고 사진 올리는 콘테스트/이벤트 (예: 아바타 콘테스트).
3. **운영 도구** — 히스토리 경품 이벤트, 추첨, 투표, 등수 정리, 마감시각 자동 컷.

## 2. 운영 주체 — self-service 가 절대 요구사항

- **허락님 (비개발자)** 이 평소 운영 / 콘텐츠 입력 / 콘테스트 진행 / 심사·발표 다 함.
- 사용자(방장) 도 봐주긴 함. 그러나 **방장 없이도 돌아가야 함** — 개발자 도움 없이 콘테스트를 만들고 마감하고 후보 선정하고 투표 결과 발표까지 self-service 어드민 UI 만으로 가능해야 함.
- 권한 모델: allow 어드민 (허락님) 은 친구들 newb 어드민과 별도. 같은 회원 ID 라도 권한은 사이트별 (백엔드 `api.dnfm.kr` 가 분리 enforce).

## 3. 디자인 톤

- **B급 감성** — 허락님이 직접 명시한 컨셉 (참고 이미지 톤 기준). 정돈된 공식 톤 ≠ allow. 약간의 위트 / 비대칭 / 손맛 OK.
- 친구들 newb 의 라이트 게시판 톤과 의도적으로 다름. **frontend 코드/디자인 시스템은 별도 repo 로 완전 격리** — 이 repo 안에서 톤·토큰 자유롭게 바꿔도 친구들 영향 0.

## 4. 콘텐츠 SSOT

- 정적 콘텐츠 (hero/프로필/철학/공지 placeholder): `src/lib/content.js`. 허락님이 보내준 정보(사진/설명/철학) 를 사용자가 1회 수기 박음. 이후는 어드민 UI 에서.
- 동적 콘텐츠 (콘테스트, 글, 사진, 투표, 경품 이력) 는 Stage 2 backend (`api.dnfm.kr`, 별도 repo) 가 필수. 정적 SSOT 만으로는 본업 못 굴림.

## 5. 초기 핵심 기능 — 아바타 콘테스트 (사용자 톡방 발췌)

첫 self-service 어드민 + 참여형 이벤트의 reference 구현.

**참가자 흐름:**
- 페이지 진입 → "참가 글쓰기" → 양식 입력:
  - 모험단명 / 모험단레벨 / 캐릭터명 / 코디 제목 / 설명
  - 코디 사진 업로드 (R2)
- 글 게시 = 콘테스트 후보 등록.

**어드민(허락님) 흐름:**
- 콘테스트 만들기 — 제목, 마감시각, 설명, 후보 정원, 투표 기간.
- 마감시각 자동 컷 — 그 이후 글은 참가로 안 침.
- 마감 후 허락님이 후보를 직접 봄 → 일부 골라 "투표 대상" 으로 승격.
- 투표 페이지 → 시청자 투표 → 등수 산정 → 결과 발표.

**시스템 요구:**
- 마감시각 정확 (서버 시간 기준 — 클라이언트 시간 신뢰 X).
- 사진 업로드 = backend (`api.dnfm.kr`) 가 발급한 R2 presigned PUT. public 차단, 다운로드는 presigned GET.
- 글 작성자 = 인증 회원 (익명 차단). 한 회원 1글 / 또는 다중 허용 정책 결정 필요.
- 투표 1회 (회원당). 중복 차단.
- 결과 등수 = 어드민이 수동 발표 (자동 산정값 + 어드민 보정 가능).

## 6. 후속 기능군

- 경품 이벤트 — 추첨 풀, 당첨자 히스토리, 중복 당첨 방지.
- 라이브 채널 / 다시보기 자동 임베드 (스트리머 플랫폼 URL 입력 → 임베드 표시).
- 공지/방송 알림 (친구들 newb 메인에 카드 1줄로 노출 — 백엔드 api 의 public endpoint 가 제공, 친구들가 fetch).

## 7. 친구들 격리

- `dnfm.kr` (newb) 은 **별도 repo (`dnfm`)**. frontend·디자인 시스템 완전 독립.
- 공유 = backend api (Stage 2 `api.dnfm.kr`) + 회원/세션 (쿠키 `.dnfm.kr`) + R2 뿐.
- 이 repo 에서 newb 코드 read/import/참고 X. cross-link 는 hardcoded URL (`https://dnfm.kr`) 만.
